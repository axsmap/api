AXS Map pledge settlement and Salesforce synchronization
=========================================================

Salesforce object
-----------------

AXS_Map_Pledge__c

Required additional field
-------------------------

Create these fields before enabling synchronization.

Pledge:
Label: AXS Map Pledge ID
API name: AXS_Map_Pledge_ID__c
Type: Text
Length: 24 or greater
Options: Unique and External ID

Campaign:
Label: AXS Map Mapathon ID
API name: AXS_Map_Mapathon_ID__c
Type: Text
Length: 24 or greater
Options: Unique and External ID

These fields provide duplicate-safe Pledge upserts and exact Campaign
matching. Campaign Name is never used as a fallback.

Environment configuration
-------------------------

SALESFORCE_LOGIN_URL=https://test.salesforce.com
SALESFORCE_CLIENT_ID=<connected-app-consumer-key>
SALESFORCE_CLIENT_SECRET=<connected-app-consumer-secret>
SALESFORCE_REFRESH_TOKEN=<integration-user-refresh-token>
SALESFORCE_API_VERSION=v64.0
SALESFORCE_PLEDGE_OBJECT=AXS_Map_Pledge__c
SALESFORCE_PLEDGE_EXTERNAL_ID_FIELD=AXS_Map_Pledge_ID__c
SALESFORCE_CAMPAIGN_EXTERNAL_ID_FIELD=AXS_Map_Mapathon_ID__c

Optional matching configuration:

SALESFORCE_CONTACT_EMAIL_FIELD=Email

Every Salesforce Campaign representing a Mapathon must contain the matching
MongoDB event ID in AXS_Map_Mapathon_ID__c before its pledges are synchronized.

Contact permission policy
-------------------------

The integration treats Salesforce Contacts as read-only:

- Donors and Mapathon participants are looked up by email.
- The integration never creates, edits, or deletes Contact records.
- If a donor Contact is not found, Donor__c is left blank and
  Donor_email__c is still populated.
- A participant Contact is required because Mapathon_Participant__c is part
  of the pledge relationship model. A missing participant Contact causes the
  pledge sync to fail for administrator review.

AWS secrets
-----------

Store Salesforce credentials in the staging API environment or AWS Secrets
Manager. Do not commit them to Git.

API workflow
------------

GET /donations/pledges/settlement-candidates
POST /donations/pledges/:donationId/close
POST /donations/pledges/:donationId/sync-salesforce

All three routes require an authenticated AXS Map administrator.

Salesforce automation still required
------------------------------------

There are currently no Flow Triggers on AXS_Map_Pledge__c and no pledge flow
was found in the Salesforce Flow inventory. Salesforce still needs an active
automation that sends the payment-request email and updates Status__c from
Calculated to Payment Requested.
